4. Страница My shops. Здесь список магазинов. Он состоит из:
	1. порядкового номера,
	2. названия магазина,
	3. описания магазина, 
	4. кнопки "Delete shop".
При удалении магазина, он будет еще около месяца висеть полупрозрачный, чтобы видно было, что он на стадии удаления. И вместо кнопки "Delete shop", будет кнопка "Restore shop". При клике на нее магазин восстанавливается. Под списком магазинов будет кнопка "Add New Shop +". В принципе, не думаю, что у аккаунта будет больше 20ти магазинов, но теоретически их можно создавать сколько угодно, поэтому стоит продумать и пагинацию.
	Если аккаунт только создали, в нем еще нет магазинов, поэтому там будет только одна кнопка "Add New Shop +".

5. Страница истории ордеров и выводов. При нажатии на ссылку магазина из My shops, мы попадаем на страницу самого магазина с историей ордеров. У ордеров есть пять статусов: 
	1. "in progress" - клиент создал ордер, но оплата еще не поступила.
    2. "ok" - оплата поступила, баланс магазина пополнен.
    3. "cancelled" - хозяин магазина отменил ордер (можно отменить только ордера со статусом "in progress")
    4. "failed" - клиент создал ордер, но по какой-то причине деньги не дошли.
    5. "claim balance" - это сообщение о том, что хозяин магазина вывел деньги с баланса.
Сама таблица состоит из следующих столбцов:
	1. id: идентификационный номер ордера (строка из 24 символов: "5b3647f2a1101f4e6ce76c2d")
	2. Currency: краткое название валюты.
	3. sum: стоимость покупки.
	4. date: дата создания ордера. Там будет число и время.
	5. status: статус ордера. Один из описанных выше.
	6. Cancel: здесь кнопка для отмены ордера. Появляется только там, где статус ордера "in progress".
	7. иконка монеты (пока на скринах нет, но это поле должно быть).
В магазине будет много ордеров, поэтому пагинация обязательна. Также нужно добавить фильтры по дате, статусу и валюте. Предполажительно выводить над таблицей как обычный селект. Ну это уже на твое усмотрение.

6. Страница баланса магазина. Таблица баланса состоит из следующих столбцов: 
	1. порядковый номер.
	2. Currency: краткое название валюты.
	3. sum: количество монет.
	4. claim: кнопка вывода средств.
	5. иконка монеты (пока на скринах нет, но это поле должно быть).
Здесь максимальное колличество строк в таблице - это колличество поддерживаемых монет системой. А их будет не более 50, поэтому пагинацию здесь делать бессмысленно.
Тут нужно сделать фильтр по валюте.

Теперь попапы:

1. Попап добавления магазина:
	1. название попапа сверху, что-то типа: "Add new shop form"
	2. поле названия магазина,
	3. поле описания магазина,
	4. кнопка отмены,
	5. кнопка отправки.

2. Попап подтверждения какого-либо действия:
	1. название попапа сверху, Например для отмены ордера: "Cancel Order"
	2. текст попапа (Are you shure, that you wont cancel this Order?)
	3. кнопка отмены (no, cancel)
	4. кнопка подтверждения действия (yes, i'm shure)

Попап подтверждения будет всплывать при следующих действиях:
	1. удаление магазина,
	2. восстановление магазина,
	3. отмена ордера.

3. Попап вывода средств:
	1. название попапа сверху ("claim balance"),
	2. поле для ввода суммы,
	3. кнопка отмены.
	4. кнопка подтверждения запроса.

Все текста писались от балбы, поэтому их можно менять, если кажется, что они подораны неудачно.
